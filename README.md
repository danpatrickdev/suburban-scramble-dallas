# Suburban Scramble: Dallas Edition

A 16-bit top-down vertical-scrolling survival shooter set in suburban Dallas.
Built with **SvelteKit 2 + Svelte 5 (runes) + Phaser 3 + TypeScript + Tailwind v4**.

## v1 status

This repo currently ships the **v1 vertical slice**: one playable level (Katy Trail — Uptown), all five characters selectable, one boss (The Main Character / Influencer), Pokémon-style dialogue, HUD, combo / Frenzy mode, and localStorage high scores. Levels 2–7 (Lower Greenville, Highland Park, Katie's Evoque Run, Fourth of July, Northlake) are designed and reserved as data slots — see the design doc and `src/lib/game/data/levels/` to add them.

## Asset pipeline

Sprites live under `static/assets/` as **Aseprite-format atlases** — each entity has a `<name>.png` (sprite sheet) and a `<name>.json` (frame layout + animation tags). All five characters and their portraits are now drawn by **real Aseprite**.

### Regenerate the 5 character sprites

```bash
/Applications/Aseprite.app/Contents/MacOS/aseprite -b --script scripts/aseprite/characters.lua
```

This drives Aseprite (v1.3.0+ must be installed) and writes PNG sheets, JSON atlases (`{tag}_{tagframe}` filename format), `.aseprite` source files, and 64×64 portraits. Edit [`scripts/aseprite/characters.lua`](scripts/aseprite/characters.lua) to adjust palettes, builds, and per-frame drawing — re-run to regenerate.

### Enemies / boss / tile

Currently still procedurally generated by [`scripts/generate-placeholders.mjs`](scripts/generate-placeholders.mjs). Either:
- **Manual Aseprite:** open and edit the PNG + JSON pair in Aseprite (export with JSON-Hash format and `{tag}_{tagframe}` filename pattern).
- **Or extend the Lua script** with `peloton`, `leash`, `coffee`, `main_character` SPECS following the same pattern as the characters.
- **Or use `pixel-plugin`:** [willibrandon/pixel-plugin](https://github.com/willibrandon/pixel-plugin) lets you generate sprites by natural-language prompt. Already installed in this repo's setup; run `/pixel-setup` after restarting Claude Code, then ask the agent to draw the missing sprites.

### Hot-swap any sprite

Drop a replacement `<name>.png` + `<name>.json` into `static/assets/<category>/` (frame names must include `idle_0..3, walk_0..3, attack_0..2, special_0..2`). Restart the dev server — no code change.

## Adding a new level

1. Create `src/lib/game/data/levels/<id>.ts` — a `LevelDef` with tile, waves, dialogue, boss key.
2. Register it in `src/lib/game/data/levels/index.ts` under `LEVELS`.
3. Optional: drop a new tile PNG in `static/assets/tiles/`.

The full game arc is laid out in the planning doc:

```
0. driveway          — tutorial, dodge sprinklers
1. katy_trail        — Uptown joggers, Influencer boss   ← v1
2. lower_greenville  — Tudor brick, brunch queue boss
3. highland_park     — Karens, HOA Drones, Pres. Sterling
4. evoque_run        — Katie-only; Range Rover Evoque driving level
5. fourth_of_july    — Night fireworks, safe zones
6. northlake         — Open-range frontier, Tornado Siren (final boss)
```

## Adding a new character

1. Add stats + visuals to `src/lib/game/data/characters.ts`.
2. Create `src/lib/game/entities/characters/<Name>.ts` extending `Player`; override `attack()` and `special()`.
3. Add a portrait + sprite sheet to `static/assets/characters/` and `static/assets/ui/portraits/`.
4. Add a switch case in `GameScene.spawnPlayer`.

## Saving / persistence

Everything is `localStorage`:

- `ssd:last_character` — last selected character
- `ssd:high_scores` — `{ <levelId>: { <characterId>: distance } }`
- `ssd:unlocked_characters` — JSON array
- `ssd:total_treats` — number

## Tooling tips for further work

- `opensrc` — `opensrc path phaser` returns a cached path to the Phaser source. Use `rg "<symbol>" $(opensrc path phaser)/src` to dig into Phaser internals.
- `npx @sveltejs/mcp svelte-autofixer "<svelte source>"` — analyzes a component for non-idiomatic Svelte 5 patterns. Useful before merging UI changes.
